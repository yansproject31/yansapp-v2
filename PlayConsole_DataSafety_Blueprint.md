This blueprint is designed to assist the legal, compliance, and engineering teams of **YANSPROJECT.ID** in successfully completing the Google Play Console **Data Safety Questionnaire**. Incorrect declarations regarding user data and permissions (especially Bluetooth connectivity, Storage, and Crashlytics) can result in app rejection or suspension. Follow the instructions below precisely to guarantee a 100% review approval rate.

---

## 🛡️ Executive Summary: Data Collection Philosophy
YANSPROJECT.ID ERP is an offline-first financial and enterprise resource planning application built for SME merchants. 
1. **Data Security:** All financial and transaction data is fully encrypted using military-grade AES-256 SQLCipher on the device and transmitted over secure HTTPS channels with SSL Pinning.
2. **Access Controls:** Only authorized personnel (OWNER or MEMBER roles) can access the app.
3. **Data Protection:** The app does not sell, lease, or distribute user data to any third-party advertising or marketing trackers. All third-party SDKs (such as Firebase Crashlytics) are purely used for operational stability and crash diagnostics.

## 📊 Questionnaire Section 1: Data Collection & Security

### 1. Does your app collect or share any of the required user data types?
*   **Answer:** **Yes** (We collect crash logs and diagnostic info via Firebase for stability, and store user credentials to perform Cloud Sync).

### 2. Is all of the user data collected by your app encrypted in transit?
*   **Answer:** **Yes**
    *   *Technical Justification:* All network communication is established using `HTTPS` (TLS 1.3/1.2) with hardcoded Certificate Pinning (Enterprise SSL Pinning) to prevent intermediate interception (MITM attacks) of sensitive transaction and invoice payloads.

### 3. Do you provide a way for users to request that their data be deleted?
*   **Answer:** **Yes**
    *   *Implementation:* Users can perform a Database Wipe inside the Settings tab (only available for the OWNER role) or request complete account erasure by contacting YANSPROJECT.ID customer support at `support@yansproject.id` or via the official web portal.

## 📁 Questionnaire Section 2: Specific Data Declarations

You must declare the following categories of data as **Collected** (and **Not Shared**):

### Category A: App Performance and Diagnostics (via Firebase Crashlytics)
*   **Data Type:** **Crash logs**
    *   *Collected:* **Yes**
    *   *Shared:* **No**
    *   *Purpose:* **App Functionality, Analytics, and Diagnostics** (Used strictly to monitor app crashes, freeze-ups, and force-closes so our engineers can fix bugs).
    *   *User Control:* Users cannot opt-out as this is a core telemetry requirement for financial stability, but it contains no Personally Identifiable Information (PII).

*   **Data Type:** **Diagnostics**
    *   *Collected:* **Yes**
    *   *Shared:* **No**
    *   *Purpose:* **App Functionality & Diagnostics** (Used to measure battery drain, render times, and database write latencies to optimize low-end Android POS terminals).

### Category B: Account & Credentials
*   **Data Type:** **User IDs & Roles** (Firebase Auth data)
    *   *Collected:* **Yes**
    *   *Shared:* **No**
    *   *Purpose:* **Account Management, Fraud Prevention & Security** (Used to distinguish access levels between OWNER and MEMBER roles, and protect sensitive financial databases from unauthorized entry).

## 📡 Questionnaire Section 3: Hardware Permissions Justification

When prompting the user for system permissions, use the legal and technical justifications below to describe the operational necessity.

### 1. Bluetooth Connection (`android.permission.BLUETOOTH_CONNECT`)
*   **Technical API:** Bluetooth ESC/POS Printing
*   **Operational Purpose:** Used to discover, connect, and stream binary receipt layouts to 58mm thermal ticket printers. This is a business-critical requirement for the In-Store POS module to print receipts for physical customers instantly.
*   **Safety Declaration:** No personal user data, location data, or contact lists are collected, stored, or transmitted via the Bluetooth radio.

### 2. External Storage Access (`android.permission.WRITE_EXTERNAL_STORAGE` / `READ_EXTERNAL_STORAGE` / Storage Scoped Access)
*   **Technical API:** `PdfDocument` Canvas Matrix Renderer
*   **Operational Purpose:** Used to render, save, and output native A4 invoice sheets as PDF files inside the device's public `Documents` folder so that users can share invoice PDFs via WhatsApp, email, or other enterprise communication channels.
*   **Safety Declaration:** The app only accesses files generated by itself. It does not scan, modify, or read any unrelated user photos, videos, or external media files.
